# 需求分析 —《汉字保卫战》

**分析框架**: /requirement-analysis 6角色  
**灵感来源**: Plants vs Zombies × 字族学习法  
**日期**: 2026-05-30  

---

## 一、核心玩法

```
植物大战僵尸              汉字保卫战

向日葵 → 阳光             学字 → 获得"墨水"
豌豆射手 → 单发攻击        基础汉字(山/水/火) → 单字攻击
寒冰射手 → 减速            偏旁字 → 属性克制
樱桃炸弹 → 范围爆炸         会意字 → 范围攻击
坚果墙 → 防御              结构字 → 阻挡

僵尸 → 靠近吃植物           错别字怪 → 靠近破坏文字塔
路障僵尸 → 更耐打           多笔画的错字怪 → 血量更多
巨人僵尸 → Boss             生僻字Boss → 需要特定字族攻击
```

## 二、游戏机制

### 2.1 资源系统
```
墨水(MoShui) = 阳光
  - 正确认字 +10 墨水
  - 连续正确 + 连击加成
  - 字族克制 + 双倍墨水
```

### 2.2 汉字塔防
```
防御塔 = 汉字卡片
  - 放置: 选已学汉字 → 拖到格子 → 自动攻击
  - 升级: 学完整个字族 → 塔升级为"字族塔"(攻击力×2)
  - 克制: 水字旁 → 克火属性怪
```

### 2.3 敌人设计
```
错别字怪: 身上写着错误汉字 → 用正确汉字消灭
形近字怪: 己/已/巳 混在一起 → 需要辨认正确字
同音字怪: 青/清/蜻 → 需要用正确偏旁字攻击
Boss: 每5关 → 需要回答3个识字题才能击败
```

## 三、学习设计

### 3.1 每关学习目标
```
关1-5:   基础高频字(人口手日月)
关6-10:  木字旁家族(树森林松柏)
关11-15: 水字旁家族(江河湖海)
关16-20: 火字旁家族(灯烧烤炎)
```

### 3.2 学习效率
```
每关时长: 3-5分钟
每关新字: 3-5个
复习旧字: 10-15个 (通过战斗)
学习密度: 比纯识字游戏高3-5倍
```

## 四、技术可行性

### React + Flame 实现
```
塔防网格:    CSS Grid 5×9
敌人移动:    Framer Motion + requestAnimationFrame
攻击检测:    行列匹配(同一行的塔攻击同一行的怪)
墨水系统:    Zustand store
关卡数据:    JSON (关数/字族/敌人数/Boss)
```

```
核心引擎(网格+塔+怪):   8h
关卡数据(20关):        4h
攻击动画+特效:         6h
UI(选塔+升级+墨水):    4h
音效:                  2h
总计:                  ~24h
```

## 五、与现有游戏的关系

```
现有:
  森林寻宝 → 探索字族(认识阶段)
  泡泡大战 → 听音辨字(识别阶段)
  配对闯关 → 图文配对(巩固阶段)

新增:
  汉字保卫战 → 实战运用(应用阶段) ← 最高层次

学习金字塔:
  听讲 → 5%  保留
  阅读 → 10%
  视听 → 20%
  演示 → 30%
  讨论 → 50%
  实践 → 75%  ← 配对闯关
  教别人 → 90% ← 汉字保卫战(用字战斗=实践+即时反馈)
```

## 六、结论

✅ 可行。一个关卡=5分钟密集学习20个字。
比泡泡大战学习效率高4倍，因为"战斗"=主动回忆。
